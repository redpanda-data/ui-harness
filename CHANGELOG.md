2026-04-23 -- Green != done warnings gate. Two new hooks (89 -> 91 wired, 98 -> 100 scripts). (1) `test-warning-check.sh` (PostToolUse Bash) scans exit-zero output from `vitest` / `bun test` / `playwright` / `tsgo` / `tsc` / `biome` / `lint` commands for a curated warning pattern set: Node `DeprecationWarning`, `ExperimentalWarning`, `(node:...) XxxWarning`, React `Warning: An update to .* inside a test was not wrapped in act`, `Warning: Each child in a list should have a unique "key"`, `Warning: validateDOMNesting`, `Warning: Failed prop type`, `Warning: Cannot update a component .* while rendering`, `Warning: Received .* for a non-boolean attribute`, `MaxListenersExceededWarning` / `PossibleEventEmitterMemoryLeak`, `UnhandledPromiseRejection` / `Unhandled Rejection` / `Unhandled Errors`, vitest `stderr |` during test, `Tests skipped N`, `[vitest] warn`, `@ts-expect-error`, `@ts-ignore`, `npm WARN deprecated`, `peer dep missing`, `playwright.*warning`, `Test ended with interrupted`. On match emits `systemMessage` nudge (suppressOutput) with first 160 chars of the matching line + remediation hint. Per-test-file escape hatch: add `// allow: test-warning` to the specific file with a reason; when ALL matching paths carry the comment the hook suppresses. Global env bypass: `TEST_WARNINGS_ALLOW=1`. Streak tracker in session dir (`warning-streak`): 3x consecutive same-kind hits appends `[3x same warning -- STOP rerunning. Read ALL, fix at source, then one clean run.]` to the nudge. Wired into PostToolUse Bash chain after `consecutive-failure-check.sh`. (2) `ci-warning-audit.sh` (Stop hook). Gates: `gh` CLI available, feature branch (not main/master/develop), session touched code, PR exists for branch, all checks SUCCESS (pending/failing/error defers to `lifecycle-stop.sh`). Resolves PR head SHA via `gh pr view --json headRefOid`, locates matching completed successful run via `gh run list --branch --limit 5 --json`, fetches `gh run view <id> --log` through a `timeout 45` guard (or `gtimeout` fallback on macOS) piped directly into the same curated warning regex, heads 30 results. Per-SHA cache file at `$_hook_session_dir/ci-warning-audit.<sha>.done` -- repeat Stops on the same commit skip the fetch. Empty results -> info log, no emit. Non-empty -> `hook_warn` (exit 0, non-blocking) with up to 6 sample lines + suggested `gh run view <id> --log | grep -E 'Warning|Deprecation|Unhandled'` fetch command. Disable via `CI_WARNING_AUDIT=0`. Wired into Stop chain between `pr-feedback-completeness-stop` and `perf-regression-stop`. (3) Docs: `CLAUDE.md` Tests section gains green-not-done rule with hook names + escape hatch syntax. `tdd/SKILL.md` `When Done` checklist adds two items: Zero warnings (local) + CI log audit. Why: passing tests can silently spew deprecations, log stderr, leak unhandled rejections -- noise that rots into real bugs when the library actually removes the API in the next major bump. "Green" was never the bar; zero warnings is. Agent must clean before calling it done. Tradeoff: nudge-only V1 (no hard blocks) -- curated allow-list beats "any word `warn`" to avoid false-positive loops on third-party library chatter that the agent cannot remediate.
